    New system

    Machines are listed when their ROM set is found in the server's /roms directory (same check the desktop version does against its roms folder). The default CPU core is the interpreter — on typical desktop workloads it is currently the fastest option here. The experimental WebAssembly recompiler is verified instruction-for-instruction against the interpreter and can be markedly faster on tight compute (number crunching, tight loops), but its per-block overhead makes Win9x desktop use slower today — tick it per system only if it measures better for your workload (the status bar shows the actual speed).

    The ⚙ button opens the card’s own options — e.g. Sound Blaster address, IRQ, DMA and OPL core — the same dialog the desktop version generates for each device.

    Attach hard-disk images on the Media tab; geometry is parsed from VHD footers automatically, or guessed from raw image sizes.

    Local files are copied into the browser-side virtual file system — nothing is uploaded anywhere. Guest writes stay in memory for this session; Download saves an image back to your computer with those changes merged in (your original file is never touched). Fixed VHDs come back as valid VHDs.

    While the machine is running: floppy and CD changes here apply instantly on save — no reboot; the guest sees a media change, so multi-disc installs just work. (The 💿 Media button in the status bar does the same without opening this dialog.) Hard-disk changes still need a reboot.

    Joystick & gamepad

    No gamepad detected — connect one and press any button on it.

    The game port itself comes with a sound card (Sound Blaster etc.), like real hardware. Connected pads map 1:1 onto the emulated sticks: left stick → axes 0/1, right stick → axes 2/3, buttons in gamepad order, d-pad → the POV hat on stick types that have one.

    MIDI output

    Sends the emulated MPU-401's stream (SB16/AWE32 — General MIDI music in DOS games) to a real or software synth via WebMIDI. After enabling, pick the output device in the sound card’s ⚙ dialog — the same “MIDI out device” row the desktop version shows.

    Network

    Frames travel over a WebSocket relay (the v86/websockproxy protocol): every machine pointed at the same relay is on the same virtual LAN — that’s the whole trick to browser-to-browser multiplayer (IPX DOOM, Windows file sharing), and the public relay also answers DHCP and NATs to the internet. It is shared and low-bandwidth; for a private LAN run node tools/net_relay.mjs and enter its ws://host:8087/ here. Tabs in this browser needs no server at all: machines in other tabs of this same browser share a LAN instantly — the quickest way to try IPX multiplayer against yourself. Each machine needs its own MAC.

    Windows 9x tip: on a PCI machine pick the Realtek RTL8029AS — it’s Plug & Play and Windows sets it up by itself. The NE2000 is a legacy ISA card (real ones were too): Windows cannot detect it — add it via Add New Hardware → “Novell/Anthem NE2000 Compatible”, then set its resources in Device Manager to I/O 300–31F, IRQ 10 (or whatever the ⚙ here says). DOS packet drivers take the same values on the command line (ne2000.com 0x60 10 0x300).
